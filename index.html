Student Webpage Tutorial

Launch your own site in 20 minutes

This tutorial guides complete beginners — no coding experience required — to create and deploy their personal Queen’s Computing student webpage.

Step 0: Sign in (or create a GitHub account)

~1–2 minutes

  • Go to GitHub and sign in. New? Click Sign up and choose the free plan.
  • Confirm your email so Pages can publish later.
  • Encouraged: Apply for the GitHub Student Developer Pack. It gives free GitHub Pro and access to GitHub Copilot Chat (AI assistant) for free!
Tips & fixes
  • No verification email? Check spam or click Resend verification.
  • Forgot password? Use Forgot password on the sign‑in page.
  • Student Pack rejected? Re‑apply with your Queen’s email and a photo of your student card.

Step 1: Create your repository

~2 minutes

  1. Click the + in the top bar → New repository.
  2. Name it queens‑student‑page and check Add a README.
  3. Choose Public, add MIT License → Create repository.
Tips & fixes
  • Repo name taken? Try queens‑student‑page‑2025.
  • Private repo? GitHub Pages works only on Public (unless paid plan).

Step 2: Add your homepage

~4 minutes

Add a file named index.html at the root (main level) of the repo and paste:

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1"/>
    <title>My Queen’s Student Page</title>
    <link rel="stylesheet" href="style.css"/>
  </head>
  <body>
    <header><h1>Welcome to My Page</h1></header>
    <main>
      <h2>About Me</h2>
      <p>I am a student at Queen’s School of Computing.</p>
    </main>
  </body>
</html>
Tips & fixes
  • Where to place? The file should be at the top level of your repo, not inside a folder.
  • Wrong name? Must be exactly index.html.

Step 3: Add Queen’s branding (style.css)

~3 minutes

Create a file named style.css (at the same level as index.html) and paste:

:root{--red:#aa1e22;--gold:#f2a900;--blue:#002d72}
body{font-family:system-ui,sans-serif;margin:0;background:var(--red);color:#fff}
header{background:var(--blue);color:#fff;padding:1rem}
main{background:#fff;color:#1a1a1a;max-width:900px;margin:2rem auto;border-radius:12px;padding:1.5rem}
a{color:var(--blue)}
Tips & fixes
  • Not working? Ensure <link rel="stylesheet" href="style.css"> is in the <head> of index.html.
  • Colors look off? Double‑check the hex codes.

Step 4: Enable GitHub Pages

~3 minutes

  1. Go to Settings → Pages.
  2. Set Source → Deploy from a branch.
  3. Choose Branch main, Folder /root → Save.
  4. Wait for the green check, then open the link.
Tips & fixes
  • 404? Wait a minute and refresh.
  • No URL? Confirm repo is Public and branch/folder are set.

Step 5: Personalize & publish updates

~6–7 minutes

You have two options:

  1. Manual edits: Open index.html and add sections: 
    <h2>Projects</h2>
<ul><li>Project 1</li></ul>

<h2>Contact</h2>
<p>Email me at student@queensu.ca</p>
    Upload images (drag to repo) and use <img src="filename.jpg" alt="me" />.
  2. With Copilot Chat: Once you activate your GitHub Student Pack, open VS Code with Copilot Chat. Ask in plain English, e.g., “Add a section to show my favorite courses styled with Queen’s colors.” Copilot will suggest HTML+CSS changes.
Tips & fixes
  • Image not showing? Must be in the repo root and path must match filename exactly.
  • Unsure how to phrase to Copilot? Be specific: “Add a gold button that links to my LinkedIn.”
  • Copilot changes in wrong place? Ask it: “Put this inside <main> under the About Me section.”
Congrats! You now have a live, customizable student page — and you can use Copilot as your assistant to expand it further.

Optional: Make changes via a Pull Request (Code Review)

bonus 5–7 minutes

  1. Edit a file, choose “Create a new branch” → name it feature/about-update.
  2. Click Propose changes → Create pull request.
  3. Review → Merge → Delete branch.
Tips & fixes
  • Merge blocked? Confirm PR targets main.
  • Conflict? Use GitHub’s Resolve conflicts editor.
